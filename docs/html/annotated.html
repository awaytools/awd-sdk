 CAttrBaseThe BaseClass for all attributes. Should never be instantiated directly
 CNumAttrThe UserAttr defines one block-property that can be added to a AWDElement that implements AttrElementBase.
Like UserAttr a NumAttr is always part of a dedicated list NumAttrList.
A NumAttr must be initiated with a default-value. It will only be exported if its value is different from the default-value.
All standart AWD-properties will be made available in constructors of the AWDElements that implements AttrElementBase.
This way, all default values for properties are set inside the libawd. If custom properties are needed, they must be created with NumAttrList.add() before the value can be set with NumAttrList.set()
