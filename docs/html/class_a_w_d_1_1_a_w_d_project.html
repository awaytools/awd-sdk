Detailed Description

The main interface into the libawd.
AWDProject holds a list of pointers to BLOCKS and a list of pointers to AWD::FILES::AWDFile.

Create a new AWDProject. There is no singleton behaviour enforced, but the libAWD was not created with multiton behaviour in mind.

Parameters
[in]new_project_typeThe TYPES::project_type for this AWDProject. Defines the inital behaviour of the AWDPoroject.
[in]initial_pathDefines the root_directory for this AWDProject. If the path contains a file-name, a AWDFile for this filename will be created. If the path contains no file-name no AWDFile will be created.
[in]generator_nameThe name of the application thats making use of the libAWD. This information will be written into the meta-dat of all created AWDFile.
[in]generator_versionThe version of the application thats making use of the libAWD. This information will be written into the meta-dat of all created AWDFile.

Adds a AWDBlock to the AWDproject. If no AWDfile is set as active-file, the function will fail. The AWDBlock must already be part of the AWDproject, otherwise the function will fail. If a AWDfile is set as active-file, the block will be added into this AWDFile. The block and all its dependencies will be handled as external assets, provided that they are not already added to the AWDFile.

Adds a AWDBlock (BLOCKS) as a shared asset to a AWDFile (and the AWDProject). Dependencies will be handled as external assets, provided that they are not already added to the AWDFile. the function will check if the block has a appearence in another AWDFile. If this is not the case, the Block_Instance will not be added.
